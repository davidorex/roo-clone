# Plan for Removing Tools from `cline` System Prompt

This document outlines the envisioned plan to remove the tools `ask_followup_question`, `plan_mode_respond`, and `new_task` (and potentially `load_mcp_documentation`) from the system prompt presented to the LLM in the `cline` codebase. This plan is based on the analysis of `cline/src/core/prompts/system.ts`.

## Background

The `cline` system prompt, generated by the `SYSTEM_PROMPT` async function in `cline/src/core/prompts/system.ts`, includes definitions for various tools available to the LLM. The goal is to modify this system prompt to exclude specific tools.

## Envisioned Plan for Tool Removal

### 1. Removing `ask_followup_question` and `plan_mode_respond`

*   **Current Location of Definitions:**
    *   These two tools are explicitly defined as multi-line string blocks within the main `# Tools` section of the `SYSTEM_PROMPT` template literal function in `cline/src/core/prompts/system.ts`.
    *   The `ask_followup_question` definition starts with the header `## ask_followup_question` and ends immediately before the `## attempt_completion` tool definition header.
    *   The `plan_mode_respond` definition starts with the header `## plan_mode_respond` and ends immediately before the `# Tool Use Examples` section header.

*   **Proposed Removal Method (Future Implementation):**
    *   The primary action will be to edit the `cline/src/core/prompts/system.ts` file.
    *   The entire text block defining `ask_followup_question` (from its `##` header up to, but not including, the `## attempt_completion` header) will be deleted.
    *   Similarly, the entire text block defining `plan_mode_respond` (from its `##` header up to, but not including, the `# Tool Use Examples` header) will be deleted.

*   **Expected Impact:**
    *   By removing these text blocks directly from the `SYSTEM_PROMPT` string, these tools will no longer be described to the LLM, effectively removing them from its available toolset as presented in the prompt.

### 2. Removing `new_task` (and potentially `load_mcp_documentation`)

*   **Current Observation:**
    *   Based on the review of `cline/src/core/prompts/system.ts` so far, the tools `new_task` and `load_mcp_documentation` do **not** appear to be defined with their own distinct `## Tool Name` sections within the main `# Tools` list where other tools like `execute_command` or `ask_followup_question` are found.

*   **Proposed Removal Method (Future Implementation):**
    *   **Step A: Investigation to Locate Definitions:**
        *   The first crucial step is to definitively identify where the definitions for `new_task` and `load_mcp_documentation` are introduced into the final system prompt that the LLM receives.
        *   **Option 1 (Within `system.ts`):** Conduct a thorough re-examination of the *entire* `SYSTEM_PROMPT` string within `cline/src/core/prompts/system.ts`. These tool definitions might exist in a different subsection (e.g., under a more general "Capabilities," "Advanced Tools," or "Other Tools" heading) that was not part of the main `# Tools` block previously focused on.
        *   **Option 2 (Programmatic Injection):** If not found directly in the `SYSTEM_PROMPT` string literal, investigate whether `cline/src/core/Cline.ts` or `cline/src/core/webview/ClineProvider.ts` programmatically augments the base system prompt. These classes might append or inject the definitions for `new_task` and `load_mcp_documentation` based on specific conditions, configurations, or by calling other utility functions. This would involve reading the source code of these files.
    *   **Step B: Removal from Identified Source:**
        *   Once the source of their definitions is pinpointed, the action will be to delete or comment out those specific definition blocks or the code responsible for their injection.

*   **Expected Impact:**
    *   Successfully removing these definitions from their source will prevent them from being included in the system prompt.

## Next Steps (for Implementation Phase)

1.  For `ask_followup_question` and `plan_mode_respond`: Proceed with editing `cline/src/core/prompts/system.ts` to delete their definition blocks.
2.  For `new_task` (and `load_mcp_documentation`):
    *   Begin by thoroughly re-reading the complete `cline/src/core/prompts/system.ts` file to search for their definitions.
    *   If not found, proceed to examine `cline/src/core/Cline.ts` and then `cline/src/core/webview/ClineProvider.ts` for any logic that might add these tool definitions to the prompt.

This plan outlines the strategy for modifying the system prompt. The feasibility and exact method for removing `new_task` depend on the outcome of the investigation phase.
