# 3. Prompt Assembly Logic

**Parent Report:** [Technical Report: `src/core/prompts/system.ts`](./00_src_core_prompts_system_report_overview.md)

The core of the standard prompt assembly happens within the internal `generatePrompt` function. This function constructs a `basePrompt` string by concatenating various sections in a predefined order. This order is critical as it dictates how information is presented to the LLM.

**Order of Assembled Sections in `basePrompt`:**

The sections are assembled as follows (referencing lines 69-101 in `src/core/prompts/system.ts`):

1.  **`roleDefinition`**:

    - Source: Determined by `promptComponent?.roleDefinition` or `modeConfig.roleDefinition`.
    - Purpose: Defines the AI's persona and primary role.

2.  **`markdownFormattingSection()`**:

    - Source: Imported from `./sections` (specifically `markdown-formatting.ts`).
    - Purpose: Provides guidelines on how the AI should format its Markdown responses.

3.  **`getSharedToolUseSection()`**:

    - Source: Imported from `./sections` (specifically `tool-use.ts`).
    - Purpose: General instructions or information regarding tool usage.

4.  **`getToolDescriptionsForMode(...)`**:

    - Source: Imported from `./tools` (specifically `tools/index.ts`).
    - Purpose: Provides detailed descriptions of all tools currently available to the AI based on its mode and context. This is a significant and dynamic part of the prompt.

5.  **`getToolUseGuidelinesSection()`**:

    - Source: Imported from `./sections` (specifically `tool-use-guidelines.ts`).
    - Purpose: Offers specific guidelines and best practices for how the AI should utilize its tools.

6.  **`mcpServersSection`**:

    - Source: Generated by `getMcpServersSection` (from `./sections/mcp-servers.ts`).
    - Purpose: Lists available Model Context Protocol (MCP) servers and their capabilities.
    - Conditional: This section is only included if the current `modeConfig.groups` contains a group named "mcp".

7.  **`getCapabilitiesSection(...)`**:

    - Source: Imported from `./sections` (specifically `capabilities.ts`).
    - Purpose: Outlines the AI's general capabilities and limitations.

8.  **`modesSection`**:

    - Source: Generated by `getModesSection` (from `./sections/modes.ts`).
    - Purpose: Describes the available operational modes for the AI.

9.  **`getRulesSection(...)`**:

    - Source: Imported from `./sections` (specifically `rules.ts`).
    - Purpose: Defines the core rules and constraints the AI must adhere to.

10. **`getSystemInfoSection(...)`**:

    - Source: Imported from `./sections` (specifically `system-info.ts`).
    - Purpose: Provides contextual system information (e.g., OS, CWD) to the AI.

11. **`getObjectiveSection()`**:

    - Source: Imported from `./sections` (specifically `objective.ts`).
    - Purpose: Clearly states the AI's primary objective for the current task or interaction.

12. **`addCustomInstructions(...)`**:
    - Source: Imported from `./sections` (specifically `custom-instructions.ts`).
    - Purpose: Appends any custom instructions provided through mode configuration, global settings, or other contextual sources (like `.rooignore` related instructions). This is typically the final major section.

Each section is separated by double newlines (`\n\n`) to ensure clear delineation in the final prompt string. The `await` keyword is used for `addCustomInstructions` and for resolving the promises from `getModesSection` and potentially `getMcpServersSection`, indicating these might involve asynchronous operations (e.g., file reading for custom instructions or mode definitions).
